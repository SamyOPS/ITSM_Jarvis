# Contributing Rules

## Workflow
- 1 issue = 1 branch = 1 pull request.
- Create a branch from `dev`.
- Open the pull request to `dev`.
- Merge `dev` to `main` when `dev` is stable.

## Branch Naming
- `feature/<name>` for new features.
- `fix/<name>` for bug fixes.

## Commit Naming
- `feat: ...` for feature work.
- `fix: ...` for bug fixes.
- `chore: ...` for maintenance/tech tasks.
- `docs: ...` for documentation changes.
